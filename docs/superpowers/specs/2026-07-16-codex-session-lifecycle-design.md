# Codex 会话生命周期设计

## 目标

让原生 macOS 版 Codex Bar 的“Codex 会话”页管理本机真实会话：归档、取消归档和彻底删除。

## 范围与边界

- 操作对象仅限 `~/.codex/sessions` 与 `~/.codex/archived_sessions` 中、已由索引验证的 JSONL 会话文件。
- 归档和取消归档复用本机 Codex CLI 的 `archive`、`unarchive` 子命令，不自行推测其迁移规则。
- 彻底删除只删除已验证会话文件；不会读取、显示或保存会话正文、认证信息或其他 Codex 数据。
- 删除不可恢复，必须在原生确认弹窗中二次确认。
- 不修改用户现有未提交的其他文件。

## 数据与服务

`SessionIndexEntry` 增加独立的 Codex thread ID 和存储状态（活跃或已归档）。`SessionIndexSource` 扫描两个受信根目录，并以文件路径作为 UI 身份，避免相同标题歧义。

新增单一 `SessionLifecycleSource`：

- `archive` 与 `unarchive` 通过 `Process` 执行官方 Codex CLI，并以 thread ID 作为参数。
- `delete` 在执行前以标准化路径确认文件位于活跃或归档根目录内，且扩展名为 `.jsonl`；不满足条件即拒绝。
- 每个操作都返回可展示的错误信息，且不吞掉命令失败。

## 界面与交互

会话页增加“全部 / 活跃 / 已归档”状态筛选。卡片展示状态徽标：

- 活跃会话显示“归档”。
- 已归档会话显示“取消归档”。
- 两类会话均显示“彻底删除”。

归档或取消归档成功后立即重新索引。彻底删除先弹出不可恢复提示，用户确认后再删除并重新索引。单个会话操作进行中时，该卡片的所有操作禁用并显示进行状态；失败时页面展示可理解的错误信息。

## 测试与验收

- 索引测试覆盖活跃和归档会话及其状态。
- 生命周期服务测试覆盖官方命令参数、删除合法文件、拒绝根目录外路径与拒绝非 JSONL 文件。
- UI 状态由数据层测试间接覆盖；在 macOS 应用中手动验证筛选、确认弹窗、成功刷新和失败提示。
- 运行 Xcode 测试目标与构建检查。
