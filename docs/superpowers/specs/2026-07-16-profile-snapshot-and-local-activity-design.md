# Profile 快照与本机活动设计

## 目标

保留 Token 活动模块，但将“全设备官方数据”和“本机 Mac 明细”明确分离。全设备总览来自用户手动导入的 Codex Profile 分享卡片；本机明细来自本机会话元数据，不再宣称与个人 Profile 的统计一致。

## 数据边界

### 全设备官方快照

- 来源：用户从 Codex Profile 选择或拖入的分享卡片截图。
- 仅保存用户确认后的结构化字段：累计 Token、峰值日、当前连续天数、最长连续天数、导入时间和来源标识。
- 图像仅用于本机一次性识别与确认，成功或取消后立即丢弃；不保存截图、头像、昵称、账号名或会话内容。
- 分享卡片未给出的字段（例如最长任务时长、逐日 Token）显示“官方快照未提供”，绝不推算或补零。

### 本机 Mac 明细

- 仅聚合 `~/.codex/sessions` 中可验证的会话元数据；页面标题和说明明确为“本机 Mac 明细”。
- 不与全设备快照相加、比较或用其填充官方字段。
- 本地数据的范围、时间和口径独立展示；如果无法可靠解释 Token 合计，则隐藏该合计而保留可验证的本机活动日期和会话时长。

## 交互流程

```text
选择或拖入 Profile 卡片
  -> 仅在内存中 OCR 提取候选数值
  -> 显示人工确认表单
  -> 用户确认后写入本应用的快照存储
  -> 刷新全设备官方总览
```

- 导入入口位于 Token 活动模块的“更新官方快照”按钮。
- OCR 结果必须可编辑；无法识别的字段保持空值并提示手动填写。
- 未导入时显示空状态：“导入 Codex Profile 卡片以显示全设备官方数据”。
- 卡片导入失败仅显示错误，不写入任何部分结果。

## 展示

- 模块顶部分为“全设备官方快照”和“本机 Mac 明细”两段。
- 官方快照展示四项分享卡片可提供的指标与同步时间；来源说明为“Codex Profile 分享卡片”。
- 本机明细仅展示与该数据源匹配的指标，带“仅此 Mac”标记。
- 所有原先声称“累计 Token 数、峰值 Token 数、连续天数、最长任务时长”为全设备精确数据的本地推算展示都移除。

## 持久化与测试

- 快照保存在本应用 Application Support 目录的独立 JSON 文件，与额度快照分开。
- 测试覆盖：快照编解码、缺失字段展示、确认前不落盘、确认后可恢复、取消/识别失败不落盘，以及本机明细不会修改官方快照。
- 手工验证：导入真实分享卡片后，确认显示的四项数值与用户的 Profile 卡片一致；重启应用后快照仍存在；原始图片不出现在应用数据目录。
